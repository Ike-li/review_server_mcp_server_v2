# AGENTS.md

## Codex Review Output

- All code reviews must create a Markdown record under `codex_review/`.
- File name format: `review_YYYYMMDD_HHMMSS.md`.
- Create one new file per review; do not overwrite previous review records.
- The review record should include:
  - review time
  - review scope
  - overall correctness verdict
  - overall explanation
  - findings list
- Even when there are no findings, still create a review record and explicitly note that no actionable findings were found.
